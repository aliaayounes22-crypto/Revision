What is OOP?
Paradigms & Core Concepts
💡
Key idea: OOP models real-world entities as self-contained objects that combine data (attributes) and behavior (methods) into one unit.
📦
Encapsulation
Bundling data + methods, hiding internals via private
🎭
Abstraction
Hiding complexity, exposing only the essential interface to users
🧬
Inheritance
Derived class gains capabilities of base class — "is-a" relationship
🔀
Polymorphism
Same interface, different behavior — "many forms" at runtime
Procedural Programming

Data & FunctionsSeparate entities

02
Classes & Objects
Blueprints and Instances
ClassBlueprint/template describing a type — specifies fields and methods
ObjectConcrete instance created from a class — has state and behavior
FieldData member — variable stored inside the object
MethodMember function — defines what the object can do
GetterAccessor method — returns private data
SetterMutator method — validates and sets private data
✅
Golden rule: Make all data fields private. Provide public getters and setters for controlled access.
public  Accessible from anywhere — inside & outside class

protected  Accessible by the class and its derived classes only

private  Accessible ONLY within the class itself
C++
class Rectangle {
private:
    float length;
    float width;
public:
    // Setter with validation
    void setLength(float len) {
        if (len >= 0) length = len;
        else cout << "Error: positive value required";
    }
    float getLength() { return length; }  // Getter
    float getArea()   { return length * width; }
};

// Usage
Rectangle box;
box.setLength(10.0);
cout << box.getArea();
03
Constructors & Destructors
Object Lifecycle
Default Constructor
No parameters — called when you write Rectangle r;
Parameterized Constructor
Takes arguments — Rectangle r(10, 5);
Copy Constructor
Initializes from another object — Distance d3(d2); — member-wise copy
⚠️
Constructor rules: Same name as class · NO return type (not even void) · Called automatically at object creation
🔄
Destructor: Named ~ClassName() · No parameters, no return · Called when object goes out of scope · Order: constructors forward, destructors reversed
C++ — Constructor order
class A {
public:
    A()  { cout << "A ";  }
    ~A() { cout << "~A "; }
};
class B : public A {
public:
    B()  { cout << "B ";  }
    ~B() { cout << "~B "; }
};
// B b;  →  Output: A B ~B ~A
04
Overloading
Method, Constructor & Operator Overloading
Function / Constructor Overloading
Multiple functions with the same name but different parameters (number, type, or order). Compiler picks the right version at compile time.
C++
int   add(int a, int b)         { return a+b; }
float add(float a, float b)     { return a+b; }
int   add(int a, int b, int c) { return a+b+c; }

// Constructor overloading
Rectangle();
Rectangle(float l, float w);
Operator Overloading
Give standard operators (+, -, *, ++, etc.) new meanings for user-defined types. Syntax: ReturnType operator+(Param)
C++
class Distance {
public:
    int feet, inches;
    Distance operator+(Distance d) {
        Distance temp;
        temp.inches = inches + d.inches;
        temp.feet   = feet + d.feet
                    + temp.inches / 12;
        temp.inches %= 12;
        return temp;
    }
};
// Distance d3 = d1 + d2;
ConceptOverloadingOverriding
WhereSame classDerived class
ParametersMust differ (type/count)Must be identical
Return typeCan differMust be same
PolymorphismCompile-timeRuntime
Keyword neededNonevirtual in base

Inheritance
Base & Derived Classes
🧬
Syntax: class Student : public Person { ... };  —  Represents "is-a" relationship. Student is a Person. Car is a Vehicle.
Single
A → B
One class inherits from one base class only
Multilevel
A → B → C
Derived class is created from another derived class — chain
Multiple
A + B → C
C++ only — one class inherits from more than one base class
Hierarchical
A → B, C, D
Multiple derived classes inherit from a single base class
Inheritance ModeBase public becomesBase protected becomesBase private
publicpublicprotectednever accessible
protectedprotectedprotectednever accessible
privateprivateprivatenever accessible
C++ — Inheritance + base constructor call
class Animal {
protected:
    string name;
public:
    Animal(string n) : name(n) {}
    virtual void speak() { cout << "..."; }
};

class Dog : public Animal {
public:
    Dog(string n) : Animal(n) {}        // call base constructor
    void speak() override { cout << "Woof! I'm " << name; }
};
07
Polymorphism & Virtual Functions
One interface, many forms
Compile-time Polymorphism
Resolved at compile time by the compiler based on the argument types.
Achieved via:
Function Overloading Operator Overloading Constructor Overloading
Runtime Polymorphism
Resolved at runtime via the virtual function table (vtable).
Achieved via:
Virtual Functions Function Overriding Base Class Pointer
C++ — Virtual functions
class Shape {
protected: int width, height;
public:
    Shape(int a=0, int b=0) : width(a), height(b) {}
    virtual int area() {          // virtual keyword here!
        cout << "Parent class area\n";
        return 0;
    }
};

class Rectangle : public Shape {
public:
    int area() override {          // override in derived
        return width * height;
    }
};

// Runtime dispatch via base pointer:
Shape *ptr = new Rectangle(5, 3);
ptr->area();  // calls Rectangle::area() — NOT Shape::area()
08
Abstract Classes & Interfaces
Blueprints of blueprints
Pure virtualvirtual int area() = 0; — declared with =0, no body in base
Abstract classAny class with ≥1 pure virtual function — CANNOT be instantiated
PurposeActs as interface/contract — defines WHAT must exist, not HOW
Derived classMUST override ALL pure virtual functions or it also becomes abstract
ExampleShape (abstract) → Rectangle, Circle, Triangle all provide area()
C++ — Abstract class
class Shape {          // abstract class
public:
    virtual float area() = 0;  // pure virtual
};

// Shape s;  → ERROR: cannot instantiate

class Circle : public Shape {
    float r;
public:
    Circle(float r) : r(r) {}
    float area() override {
        return 3.14 * r * r;
    }
};

Friend Functions & Classes
Trusted external access
Friend Function
What it isDefined outside the class but has access to ALL private/protected members
Has this?NO — it is not a member function
Declared withfriend keyword inside the class
Use caseOperator overloading, external helper functions
Friend Class
What it isEntire class B has access to A's private/protected members
Mutual?NO — must declare friendship in both directions separately
Inherited?NO — friendship is NOT inherited
WarningUse sparingly — breaks encapsulation principle
